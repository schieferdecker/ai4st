Abstract

The AI for software testing ontology ai4st can be used to classify contributions from research papers on using AI techniques to evolve and/or improve software testing activities and processes.

Introduction back to ToC

The ai4st ontology enables the classification of AI for software testing for software testing along the following lines: the purpose of the solution; the software testing target being addressed; the type of AI being used; and the level of automation supported by AI. It uses the software testing concept ontology stc, which is a formalized ontology of the ISTQB glossary. An initial set of 38 pieces of research on AI for software testing has been classified in ai4st.

ai4st: The AI for ST taxonomy: Description back to ToC

ai4st is a taxonomy for research papers on the use of artificial intelligence (AI) methods and techniques to support software testing (ST). It relates papers to their topics and contributions. These contributions are categorised by the purpose of the solution aimed for, the ST activity or process targeted, the type of AI used and the level of automation supported.
